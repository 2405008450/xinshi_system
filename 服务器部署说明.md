# 服务器部署说明

## 服务器信息
- IP: 43.160.215.225
- 部署方式: Docker后端 + Nginx前端

## 步骤一：本地构建前端

在本地项目目录执行：
```bash
cd frontend
npm install
npm run build
```

## 步骤二：上传代码到服务器

将整个项目目录上传到服务器：
```bash
scp -r ./* user@43.160.215.225:/home/ubuntu/xinshi_system/
```

## 步骤三：数据库迁移（如需要）

1. 导出本地数据库（如未导出）：
```bash
pg_dump -U postgres -d xinshi_system > xinshi_system.sql
```

2. 上传到服务器：
```bash
scp xinshi_system.sql user@43.160.215.225:/tmp/
```

3. 服务器上启动数据库并导入：
```bash
cd ~/xinshi_system
docker-compose up -d postgres
docker exec -i xinshi_postgres psql -U postgres -d xinshi_system < /tmp/xinshi_system.sql
```

## 步骤四：启动后端Docker

```bash
cd ~/xinshi_system
docker-compose up -d --build
```

## 步骤五：配置Nginx前端

1. 安装nginx：
```bash
sudo apt update
sudo apt install nginx
```

2. 复制前端构建产物到nginx目录：
```bash
sudo cp -r ~/xinshi_system/frontend/dist/* /var/www/html/
```

3. 配置nginx代理：
```bash
sudo cp ~/xinshi_system/nginx.conf /etc/nginx/sites-available/default
sudo systemctl restart nginx
```

## 步骤六：验证

- 前端: http://43.160.215.225
- 后端API: http://43.160.215.225/api/users
- 数据库健康检查: http://43.160.215.225/api/health/db

## 常用命令

```bash
# Docker后端
cd ~/xinshi_system
docker-compose up -d          # 启动
docker-compose logs -f        # 查看日志
docker-compose restart        # 重启

# Nginx
sudo systemctl restart nginx  # 重启nginx
sudo nginx -t                # 检查配置
```

## 更新部署

如果代码有更新：

```bash
# 前端更新
cd ~/xinshi_system/frontend
npm run build
sudo cp -r dist/* /var/www/html/

# 后端更新
cd ~/xinshi_system
docker-compose build --no-cache
docker-compose up -d
```
