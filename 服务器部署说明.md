# 服务器部署说明

## 服务器信息
- IP: 43.160.215.225
- 部署方式: Docker (全部容器化)

## 第一步：本地数据库导出

在本地执行：
```bash
pg_dump -U postgres -d xinshi_system > xinshi_system.sql
```

## 第二步：上传代码到服务器

将整个项目目录上传到服务器，例如：
```bash
# 方法1: 使用scp
scp -r ./* user@43.160.215.225:/home/user/xinshi_system/

# 方法2: 使用git
git clone <your-repo> /home/user/xinshi_system/
```

## 第三步：数据库迁移

1. 上传sql文件到服务器：
```bash
scp xinshi_system.sql user@43.160.215.225:/tmp/
```

2. 进入服务器，启动数据库容器：
```bash
cd /home/user/xinshi_system
docker-compose up -d postgres
```

3. 等待数据库启动完成，然后导入数据：
```bash
docker exec -i xinshi_postgres psql -U postgres -d xinshi_system < /tmp/xinshi_system.sql
```

## 第四步：启动所有服务

```bash
docker-compose up -d
```

## 第五步：验证

- 前端: http://43.160.215.225
- 后端API: http://43.160.215.225/api/users
- 数据库健康检查: http://43.160.215.225/api/health/db

## 常用命令

```bash
# 查看日志
docker-compose logs -f

# 查看容器状态
docker-compose ps

# 重启服务
docker-compose restart

# 停止服务
docker-compose down

# 重新构建
docker-compose build --no-cache
```

## 目录结构

```
xinshi_system/
├── Dockerfile              # 后端Dockerfile
├── docker-compose.yml      # 容器编排
├── requirements.txt        # Python依赖
├── main.py                # FastAPI入口
├── database.py            # 数据库配置
├── models.py              # SQLAlchemy模型
├── schemas.py             # Pydantic模型
├── routers/               # API路由
├── frontend/

│   ├── nginx.conf         # Nginx配置
│   ├── src/               # Vue源代码
│   └── dist/              # 构建产物(自动生成)
└── xinshi_system.sql      # 数据库导出文件(本地导出)
```

## 端口说明

| 端口 | 服务 | 说明 |
|------|------|------|
| 80   | Nginx | 前端Web |
| 8000 | FastAPI | 后端API |
| 5432 | PostgreSQL | 数据库 |

## 注意事项

1. 首次部署需要先启动数据库并导入数据
2. 如果数据库连接失败，检查`docker-compose.yml`中的环境变量
3. 前端请求`/api`开头时会自动代理到后端
